## 三、目标用户
核心用户：**1-20人小型创业公司创始人、CEO、项目负责人**
核心使用场景：对接投资人、商务KOL洽谈、商务合作对接、高端人才寻访

## 四、用户核心痛点
创始人想要精准触达目标人脉时，需要手动翻阅领英、推特、油管、个人网站等平台，花费**2-3小时**拼凑联系方式与活跃渠道；
该需求使用频次低，不愿专门招聘BD与调研助理，大量时间消耗在低价值信息搜集工作中。

## 五、核心体验假设
1. 人群：1-20人规模公司创始人/CEO
2. 场景：需要触达潜在合作方、KOL、候选人、投资人
3. 现存痛点：手动全网搜集信息效率极低，无专职人员协助调研
4. 产品能力：输入人名+基础背景，AI自动完成全网深度调研，输出结构化人物档案
5. 用户行为转变：像写备忘录一样下达调研指令，次日直接依据结果确定触达渠道并落地执行
6. 产品定位：创始人专属**一句话启动的临时AI研究员**

## 六、产品核心价值
创始人时间是企业最高成本，将人脉信息搜集等低效工作交由AI完成，轻量化、低成本完成全网人脉调研，直接输出可落地商务触达方案。

## 七、用户核心主流程
### 7.1 访问入口流程
1. 用户通过搜索引擎、社交平台种草链接进入官网首页
2. 首页**无强制登录注册弹窗**，零门槛进入
4. 支持**匿名状态直接输入调研需求**，无需注册登录

### 7.2 核心操作流程
1. 用户输入完整调研需求（姓名、职位、调研诉求、输出格式）
2. AI不直接执行任务，主动弹出**两项需求确认选项**
    - 调研档位：快速版（5分钟，基础信息+主流社交账号）/ 完整版（10-15分钟，全维度分析+精准触达策略）
    - 文档存储：新建Notion文档 / 指定已有Notion文档
3. 用户选定选项后，AI展示完整调研执行计划，罗列全部搜索、分析、整理动作
4. 页面展示操作按钮：`开始构建` / `我想调整`
5. 匿名用户点击【开始构建】，唤起**悬浮登录浮层**，不跳转页面，保留全部对话内容
6. 登录完成后自动唤起Notion授权弹窗，授权通过正式启动调研任务

### 7.3 任务执行流程
1. 页面展示轻量化任务进度卡片
2. 卡片默认仅展示最新运行状态，支持点击展开查看完整运行日志
3. 卡片展示内容：任务名称、预估剩余时长、任务暂停按钮
4. 支持后台离线运行，用户可自由离开当前页面
5. 任务完成后修改浏览器标签状态，开启桌面浏览器通知可接收完成提醒

### 7.4 结果输出流程（三层信息层级）
1. 第一层（页面直显）：信息速览、联系方式数量、社交平台数量、TOP3最优触达渠道排序
2. 第二层（跳转入口）：一键跳转查看中英双语结构化Notion完整人物档案
3. 第三层（可选查看）：AI全网检索全流程运行日志
4. 结果页操作按钮：满意、补充定向调研、套用模板调研新人物

## 八、产品交互硬性规则
### 8.1 匿名用户权限边界
- 允许操作：访问首页、输入调研需求、查看AI反问、查看执行方案
- 禁止操作：启动任务构建，点击构建强制触发登录
- 登录规则：全程无页面刷新，历史对话完整保留，无需二次输入需求

### 8.2 多区域登录体系
1. 国内登录方式：微信登录、手机短信验证码、APP扫一扫登录
2. 海外登录方式：Google一键登录、Apple登录、邮箱验证码登录

### 8.3 进度卡片交互规范
1. 默认精简折叠展示，不挤占页面视觉空间
2. 任务时长超3分钟，自动提示用户可离线等待任务通知
3. 支持随时暂停、终止正在执行的调研任务

### 8.4 结果复用规则
1. 调研完成后，支持一键复用同款调研模板
2. 预留高阶入口：自定义定时批量人脉调研任务（V0仅预留）

## 九、用户转化漏斗
1. 匿名访问首页 → 零门槛输入调研需求（完全免费）
2. 确认调研方案 → 触发登录动作（用户已认可产品价值）
3. 新用户福利：首次完整调研免费，赠送体验额度
4. 付费触发节点：二次调研、批量任务调用，开放余额充值与付费体系
5. 设计逻辑：用户验证产品价值后再产生付费意愿，提升整体转化率

## 十一、V0 版本暂不开发功能
1. 人物档案日历日程自动推送
2. 大批量名单全自动批量调研
3. 自定义筛选调研数据源
4. 企业团队多人协同使用功能
5. 人脉数据深度统计分析报表

## 十二、非功能性需求
1. 页面风格：极简商务风，优先适配桌面端使用场景
2. 交互规范：所有弹窗采用浮层形式，禁止页面跳转与强制刷新
3. 体验要求：实时展示运行状态，杜绝空白加载卡顿
4. 兼容要求：适配主流PC端浏览器

## 十三、测试验收标准
1. 全流程走通：匿名输入→AI需求确认→登录授权→任务执行→结果输出完整闭环
2. 权限逻辑正常：匿名账号无法启动调研构建任务
3. 进度展示、后台离线运行、消息提醒功能运行正常
4. 调研结果三层信息层级展示符合设计规范
5. 海内外双端登录入口展示、调用逻辑无误
